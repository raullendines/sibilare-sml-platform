# Repository As AI Context

This repo is intentionally structured so AI agents can work with less ambiguity.

## Key Context Files

- `AGENTS.md`: mandatory agent rules.
- `README.md`: setup and project overview.
- `docs/product/`: product intent and business rules.
- `docs/architecture/`: technical decisions and patterns.
- `docs/decisions/`: architecture decision records.
- `guardrails/`: hard rules and review checklist.
- `.github/`: issue and pull request templates.

## Principle

The repo should document not only code, but the reasoning that prevents bad code:

- Why the system exists.
- How branches work.
- What must not be bypassed.
- Where business logic belongs.
- How to verify changes.
